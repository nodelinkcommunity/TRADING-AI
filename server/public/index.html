Guidelines & Best Practices
Follow these guidelines to maximize profit and minimize risk. This is not financial advice — always do your own research.
1

Always Start on Testnet

Deploy to Arbitrum Sepolia first. Run bots for at least 48 hours in monitor-only mode (Auto Execute = OFF) to verify opportunity detection, gas estimation accuracy, and that the bot handles edge cases like reverted transactions, stale prices, and network congestion.

2

Use a Dedicated Hot Wallet

Never use your main wallet. Create a fresh MetaMask wallet exclusively for the bot. Fund it only with the amount you are willing to lose. If the private key is compromised, your main holdings remain safe. Rotate keys periodically.

3

Understand Gas Economics

Every trade costs gas. On Arbitrum, gas is cheap (~$0.01-$0.05 per tx), but on Ethereum mainnet it can be $5-$50+. Always factor gas into your minimum profit threshold. A profitable arbitrage on paper can become a loss after gas. Use the Max Gas setting to cap spending.

4

Flashloan Mechanics

Flashloans from Aave V3 must be repaid within the same transaction block. If the arbitrage fails (slippage, frontrunning, stale price), the entire transaction reverts and you only lose the gas fee. This makes flashloans capital-efficient but not risk-free — gas costs still apply and repeated failures compound.

5

MEV & Frontrunning Protection

On-chain arbitrage is a competitive game. MEV bots and block builders watch the mempool. Use private RPCs (Flashbots Protect, MEV Blocker) when moving to mainnet to avoid sandwich attacks. On Arbitrum, MEV is less of an issue due to the FCFS sequencer model, but this may change.

6

Risk Management Rules

Set a daily loss limit and stick to it. If the bot hits 5 consecutive failed transactions, auto-pause and investigate. Never run New Pool Sniping or Oracle Lag on mainnet without thorough backtesting. Start with conservative parameters (Min Profit > 30 bps, Max Slippage < 30 bps) and tighten as you learn.

7

Monitoring & Alerts

Set up Telegram notifications for all trades. Check the Analytics panel daily. Watch for patterns: if Opportunities is high but Trades is zero, your slippage or gas settings may be too tight. If gas consumption spikes with no profit, an external MEV bot may be frontrunning you.

8

Mainnet Migration Checklist

Before going live: (1) Testnet ran 72+ hours with no critical errors. (2) Contract verified on Arbiscan. (3) Auto Execute tested on testnet with real flashloans. (4) Dedicated wallet funded with 0.01 ETH for gas. (5) Telegram alerts configured. (6) PM2 auto-restart confirmed. (7) VPS has at least 1GB RAM and stable internet.

Security Reminders
⚠
Never share your private key. This application stores it server-side in the .env file. If your VPS is compromised, your wallet is compromised. Use firewall rules to restrict access to port 3000 (IP whitelist recommended). Consider using HTTPS with a reverse proxy (Nginx + Let's Encrypt) for production deployments.
🔒
Recommended VPS Hardening: Disable root SSH login, use SSH keys instead of passwords, configure fail2ban, enable UFW firewall, and keep the system updated. Restrict port 3000 to your IP: ufw allow from YOUR_IP to any port 3000.

1. DEX-to-DEX Arbitrage
Buy low on one DEX, sell high on another within the same flashloan transaction. The bread-and-butter of on-chain arbitrage. Works across Uniswap V3, SushiSwap, Camelot, and more.
How It Works
  • Monitor the same token pair across multiple DEXs simultaneously
  • When price diverges beyond the profit threshold, take a flashloan
  • Buy the token on the cheaper DEX, sell on the expensive one
  • Repay the flashloan + fee, keep the spread as profit
Key Parameters
  • Pairs: WETH/USDC, WETH/USDT, WETH/ARB (configurable)
  • DEXs Scanned: Uniswap V3, SushiSwap, Camelot, TraderJoe
  • Flashloan Fee: 0.05% (Aave V3)
  • Typical Spread: 0.1% - 0.5% per opportunity

2. Triangular Arbitrage
Exploit pricing inconsistencies across three token pairs on the same or different DEXs. A → B → C → A, ending with more A than you started. More complex but harder for competitors to spot.
How It Works
  • Flashloan Token A (e.g., USDC)
  • Swap A → B (e.g., USDC → WETH on Uniswap)
  • Swap B → C (e.g., WETH → ARB on SushiSwap)
  • Swap C → A (e.g., ARB → USDC on Camelot)
  • If A_out > A_in + fees, profit is captured
Key Parameters
  • Triplets: WETH/USDC/DAI, WETH/ARB/USDC (configurable)
  • Gas Cost: ~2-3x a simple swap (3 hops)
  • Edge: Harder to detect = less competition
  • Complexity: Requires accurate multi-hop price simulation

3. Liquidation Sniping
Monitor Aave V3 and Compound borrower positions. When a position's Health Factor drops below 1.0, liquidate it and collect the liquidation bonus (5-15% of collateral). High reward, event-driven.
How It Works
  • Discover borrowers by scanning debt token Transfer events
  • Continuously monitor each borrower's Health Factor
  • When HF < 1.0, the position is liquidatable
  • Flashloan the debt token, repay the borrower's debt
  • Receive collateral at a discount (the liquidation bonus)
  • Swap collateral back to repay the flashloan
Key Parameters
  • HF Threshold: Monitor when HF drops below 1.1 (configurable)
  • Min Bonus: Only liquidate if bonus > 5% (configurable)
  • Protocols: Aave V3 (primary), Compound V3
  • Trigger: Market crash, sudden price drops

4. Stablecoin Depeg Arbitrage
When stablecoins trade below $1.00 (or above $1.00) on DEXs, buy the discounted stablecoin and redeem or swap it at peg. Low risk, frequent micro-opportunities, especially during market volatility.
How It Works
  • Monitor 6+ stablecoins: USDC, USDT, DAI, FRAX, MIM, USDCe
  • Detect when any stablecoin trades at >10 bps from $1.00
  • Flashloan the stronger stablecoin, swap to the depegged one
  • Wait for re-peg or swap back via a different route for instant profit
  • Includes trend analysis to predict depeg deepening
Key Parameters
  • Depeg Threshold: 10 bps (0.1%) deviation triggers alert
  • Assets Monitored: USDC, USDT, DAI, FRAX, MIM, USDCe
  • Flash Amount: $50,000 default (configurable)
  • Round-trip profit calculation includes all fees

5. New Pool Sniping
Detect newly created liquidity pools on DEX factories within seconds and execute the first buy before the crowd. Extremely high reward potential, but also high risk due to rug pulls, honeypots, and low liquidity.
How It Works
  • Listen to PairCreated events on Uniswap/SushiSwap factory contracts
  • Instantly analyze the new token: verify contract source, check for honeypot patterns, assess initial liquidity
  • If it passes safety checks, execute a small buy within the first block
  • Set take-profit and stop-loss levels automatically
  • Auto-sell at target or time-based exit
Key Parameters
  • Max Buy: $100 per new pool (configurable, risk cap)
  • Auto Approve: OFF by default (requires manual review)
  • Safety Checks: Verified source, no mint(), no blacklist()
  • Exit Strategy: 2x take-profit or 50% stop-loss

6. Oracle Lag Exploitation
Price oracles (Chainlink, Pyth) update on a heartbeat or deviation threshold. Between updates, the on-chain price can lag behind the real market price. Trade against stale oracle prices before they refresh.
How It Works
  • Monitor Chainlink price feed's last update timestamp
  • Simultaneously check real-time DEX prices (or CEX via API)
  • When oracle price diverges from true market price by >threshold
  • Borrow/lend on the protocol using the stale oracle price
  • Unwind position after oracle updates for profit
Key Parameters
  • Oracle Sources: Chainlink, Pyth, Band Protocol
  • Lag Detection: Compare on-chain vs off-chain prices
  • Heartbeat: Chainlink updates every 3600s or 0.5% deviation
  • Auto Execute: OFF by default (high complexity)

7. Yield Rebalancing
Continuously monitor lending rates across Aave, Compound, and other protocols. When a significant APY gap appears, withdraw from the lower-yield protocol and deposit into the higher-yield one using a flashloan for atomic rebalancing.
How It Works
  • Query supply APY across Aave V3, Compound V3, Morpho, etc.
  • Compare rates for the same asset (e.g., USDC on Aave vs Compound)
  • When APY difference exceeds threshold (e.g., 1%+)
  • Flashloan to atomically withdraw from A and deposit into B
  • Net gain is the APY differential minus flashloan fee
Key Parameters
  • Min APY Difference: 1.0% (configurable)
  • Protocols: Aave V3, Compound V3, Morpho, Spark
  • Assets: USDC, USDT, DAI, WETH
  • Rebalance Frequency: Check every 30 minutes
